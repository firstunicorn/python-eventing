docs(tests): add comprehensive inline documentation for bug fixes and linter exclusions

## Starting point: missing documentation

Code had fixes for critical bugs and linter exclusions but lacked inline explanations of:
- WHY bugs occurred (root causes)
- WHY specific linter checks were suppressed
- WHAT symptoms manifested during failures
- HOW solutions resolve the underlying issues

This made the codebase appear unmaintained and decisions opaque.

## Changes

### 1. Documented critical Kafka consumer race condition fix (tests/conftest.py)
- Added comprehensive explanation of async_client_with_kafka fixture's 5-second wait
- Documented bug history: QueueEmpty failures in CI due to Kafka consumer not fully joined
- Explained root cause: FastStream broker.start() returns before consumer group rebalancing completes
- Rationale: GitHub Actions containers need 5s buffer (vs 2-3s typical rebalancing time)

### 2. Documented Pydantic settings override bug (tests/conftest.py)
- Explained why monkeypatch.setenv() failed with env_prefix="EVENTING_"
- Documented root cause: Pydantic loads settings at import time before monkeypatch runs
- Solution rationale: setattr() directly mutates singleton after Pydantic instantiation

### 3. Documented SQLAlchemy session management bug (src/messaging/main/_initialization.py)
- Added detailed explanation of per-message session instantiation
- Bug history: AttributeError with async_sessionmaker vs AsyncSession confusion
- Root cause: SqlAlchemyProcessedMessageStore expects AsyncSession, not sessionmaker
- Solution: Instantiate store+consumer per message within session context
- Transaction semantics: claim → publish must be atomic (session.begin())
- Explained Ruff SIM117 compliance (combined `async with` statement)

### 4. Documented RabbitMQ exchange type bug (src/messaging/infrastructure/pubsub/rabbit/publisher.py)
- Explained implicit DIRECT exchange vs explicit TOPIC exchange issue
- Symptoms: Publish succeeded but queue.get() timed out (routing mismatch)
- Root cause: FastStream defaults to DIRECT when exchange is bare string
- Solution: Explicitly construct RabbitExchange with type=TOPIC, durable=True

### 5. Documented test UUID fixes (tests/integration/test_kafka_rabbitmq_bridge.py)
- Explained hardcoded event_id collision issue across test matrix runs
- Root cause: Idempotency store claims by event_id caused false negatives
- Solution: uuid4() per test invocation prevents collisions
- Documented CI timeout increases (15s sleep, 30s queue.get timeout)
- Measured latency breakdown: Kafka poll (1-2s), DB query (500ms), RabbitMQ (500ms) in CI

### 6. Documented linter exclusions with rationale

#### SQLAlchemy ORM models (src/messaging/infrastructure/kafka_dlq/orm_models.py, setup.cfg)
- Added docstring explaining PF002 suppression
- Rationale: mapped_column(default=...) is SQLAlchemy syntax, not Pydantic Field()
- flake8-pydantic plugin incorrectly flags SQLAlchemy Mapped[] columns as violations
- Decision: Suppress per-file in setup.cfg rather than disable plugin globally

#### FastStream middleware signature (broker_config/_factory_helpers.py, rabbit_broker_config/_factory_helpers.py)
- Documented ARG001 suppression for msg/context parameters
- Rationale: FastStream requires exact (msg, context) factory signature for dependency injection
- Cannot remove parameters even though unused (framework interface requirement)
- Reference: FastStream middleware documentation

#### Deptry DEP002 for structlog (pyproject.toml)
- Added comprehensive explanation of structlog exclusion
- Rationale: Production code is logging-library-agnostic (stdlib logging only)
- structlog ONLY used in tests/conftest.py and scripts/cleanup_docker.py
- Deliberate architectural decision: tests use structured logging, prod does not

#### Test exclusions (setup.cfg)
- Documented WHY tests have different standards (DAR*, E501, PF*, CCR001)
- pytest fixtures use implicit params from decorators (not in docstring)
- Test complexity is acceptable (sequential/conditional by nature)
- Long assertion messages prioritized over line length

### 7. Fixed structlog deptry violation
- Added structlog to DEP002 ignore list with architectural rationale

### 8. Fixed blank line at EOF bug (tests/conftest.py state update)
- Documented state=None guard to prevent TypeError
- Explained why lifespan_context() can yield None
